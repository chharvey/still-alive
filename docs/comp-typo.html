The Subhead Component

Put a .c-Subhead sibling adjacent to a heading to create a subheading for a page. This can be thought of as a tagline or subtitle for the top-level heading.

The Folio Title Component

On special pages, the top-level header may be a .c-FolioTitle. To add a little spice, use .c-FolioTitle__Cap on capital letters. The size will make them stand out.

The Lede and Drop Cap Components

The Caption Component

The Caption Component may be used on a table caption, a figure caption, a details summary element, a fieldset legend element, or simply just a paragraph describing an arbitrary element.

Subclasses .c-Caption--before and .c-Caption--after add a separating border. They’re really only good for when the figure contains text (as opposed to a table or image) and the caption text needs to be separated. The subcomponent .c-Caption__Title serves as a label.

The Pull Quote Component

A pull quote has a stronger effect than unstyled text, even if it does belong in a blockquote element. To cite the source, add a child .c-Pullquote__Src Component.

Use this Component when the original source text is less than 40 words and contains no paragraph breaks. Do not include child p elements. If the original source is long enough and/or has paragraph breaks, you shouldn’t be using this large component.

The Action List Component

Action lists are used specifically for short, one-line, textual hyperlinks.

The Footnotes List Component

If you have footnotes in the footer of a document or section, use a Footnotes List Component. Read my technical report on footnotes for more information.

Components for Tables

Use the following classes for extra styles on table and dl elements.

border around the entire table
borders in between rows
borders in between cells in rows
background shading to the entire table
background shading to alternating rows
background shading to alternating cells in rows
background shading to a row on hover

Notice: These Components are all independent of each other, that is, the use of one does not require another. For example, a table may be classed .c-TableBorderedRows but may not necessarily be classed .c-TableBordered.

The Document Component

Use .c-Document to mark up a document such as an article or blog post. A document may take the form of a body > main or body >> article. It should have a status and be marked up with metadata. Read my technical report on document status for more information. Any element inside a .c-Document may be marked with subcomponent classes indicating their status.

A Document is normative by default, but may also have informative (non-essential) elements, which typically have the boolean [data-informative] attribute.

Document Meta

.c-Document__Meta marks up meta-information about a document.

Read my technical report on document headers for more information.

Text-Level Components

Acronyms

When an abbr element contains an acronym, with all caps, use class .c-Acro for a friendlier style. It lessens the harshness of sequential capital letters.

Use .c-Acro__First on the first letter of an .c-Acro that begins a paragraph, sentence, or list item. It simply reverts the font size.
